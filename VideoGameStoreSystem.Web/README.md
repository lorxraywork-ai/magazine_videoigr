# Информационная система магазина видеоигр

Учебное ASP.NET Core MVC приложение на .NET 8 для курсовой работы по теме автоматизации магазина видеоигр.

## Технологии

- ASP.NET Core MVC (.NET 8)
- Razor Views
- EF Core + SQL Server
- Cookie Authentication
- Bootstrap 5
- Chart.js
- ClosedXML

## Структура проекта

- `Controllers` — контроллеры разделов
- `Models` — сущности предметной области
- `ViewModels` — модели представлений и фильтров
- `Data` — `DbContext`, миграции и инициализация данных
- `Infrastructure` — авторизация, хэширование, toast, экспорт, складская логика
- `Permissions` — роли и матрица прав
- `Views` — Razor-представления
- `sql` — SQL-скрипты для ручного создания БД и схемы

## Подключение к БД

Основная строка подключения уже настроена в `appsettings.json` и `appsettings.Development.json`:

`Server=mssql.lorxray.ru,14330;Database=magazine_videoigr;User Id=sanya;Password=sanya12345;TrustServerCertificate=True;Encrypt=True;`

## Подготовка базы данных

### Вариант 1. Через EF Core

1. `dotnet restore`
2. `dotnet build`
3. `dotnet dotnet-ef database update --project ./VideoGameStoreSystem.Web.csproj --startup-project ./VideoGameStoreSystem.Web.csproj`
4. `dotnet run --launch-profile http`

При первом запуске приложение:

- применяет миграции;
- создает роли;
- добавляет демонстрационных пользователей;
- заполняет категории, товары, поставщиков, клиентов, поступления, продажи и списания.

### Вариант 2. Ручное создание через SQL

1. Выполнить `sql/01_create_database.sql` в базе `master`
2. Выполнить `sql/02_create_schema.sql` в базе `magazine_videoigr`
3. Запустить приложение командой `dotnet run --launch-profile http`

После запуска приложение автоматически добавит демонстрационные данные, если таблицы пока пусты.

## Ключевые особенности

- ручная cookie-аутентификация без Identity UI;
- пароли хранятся только в виде PBKDF2-хэшей;
- централизованная матрица ролей и прав;
- обновление claims при изменении роли, логина или ФИО;
- блокировка сессии для отключенного пользователя;
- аудит `Create / Update / Delete` в `APP_CHANGE_LOG`;
- учет остатков товара при поставках, продажах и списаниях;
- запрет продажи и списания при недостаточном остатке;
- экспорт основных таблиц и отчета в Excel;
- аналитический отчет с диаграммами на Chart.js.

## Резервное копирование

Для учебной эксплуатации можно использовать простую административную процедуру резервного копирования SQL Server:

```sql
BACKUP DATABASE [magazine_videoigr]
TO DISK = N'C:\Backup\magazine_videoigr.bak'
WITH INIT, FORMAT;
```

Восстановление выполняется стандартной командой `RESTORE DATABASE`. Перед выполнением команды нужно убедиться, что путь сохранения доступен серверу SQL Server.
